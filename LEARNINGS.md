## 📅 [2026-02-13] Multi-City Flight Search UI Overhaul

**Objective:** Refactor the Multi-City search interface on the homepage to match the premium "One Way" aesthetic and resolve critical usability bugs (duplicate inputs, layout shifts).

### 🔍 Key Learnings & Solutions

#### 1. The "Ghost" Flight 1 Box (Duplicate Input Issue)

**Problem:**
When switching to "Multi-City" mode, users saw two flight boxes: the existing static search box _plus_ a new "Flight 1" box generated by the Alpine.js loop. This created a confusing, cluttered interface.

**Solution:**

- Instead of hiding the static box, we **repurposed** it.
- We used Alpine.js dynamic binding to change the input `name` attributes based on the `tripType`.
  - **One Way/Round Trip:** `name="from"`, `name="to"`
  - **Multi-City:** `name="segments[0][from]"`, `name="segments[0][to]"`
- We modified the `x-for` loop to strictly render segments starting from `index > 0` (Flight 2+), effectively merging the static box into the multi-city flow as "Flight 1".

#### 2. Managing Alpine.js State Persistence

**Problem:**
The form would sometimes retain old state or render incorrectly on reload because the component state wasn't cleanly reset.

**Solution:**

- We added `x-init="segments = [{ from: '', to: '', date: '', id: 1 }]"` to the parent `x-data` object in `welcome.blade.php`.
- This forces a clean state initialization on every page load, preventing "ghost" data from interfering with the UI.

#### 3. Handling Visual Overlap in Dense Forms

**Problem:**
The "Add Another Flight" button was physically overlapping the "Depart" date input field above it, making the UI look broken.

**Solution:**

- **Explicit Margin/Padding:** Relying on generic `space-y` utilities was insufficient for this specific layout.
- We added specific classes:
  - `mb-8` to the main search grid container.
  - `mt-8` to the multi-city container.
  - `pt-6` to the button's wrapper div.
- **Lesson:** For complex forms with dynamic visibility, explicit margins are safer than relying solely on flex gap or space utilities.

### 🛠 Technical Implementation Details

- **File:** `resources/views/welcome.blade.php`
- **Tech Stack:** Alpine.js (for state management) + Tailwind CSS (for layout).
- **Core Logic:**
  ```javascript
  // Dynamic binding example
  :name="tripType === 'multicity' ? 'segments[0][from]' : 'from'"
  ```
